# 별 수가르기 — 프로젝트 맥락 (HANDOFF)

## 무엇인가
초등 1학년(우리 딸) 용 **수 가르기·모으기** 학습 PWA. 단일 `index.html` 정적 사이트.
계기: 와이프가 외부 블로그 학습지(수 가르기, A4 9문제)를 출력해 풀게 하던 것을, **무한정 생성 + 태블릿 풀이**가 되는 자체 앱으로 만든 것. `byeol-sudoku`(별 사냥 스도쿠)의 자매 앱(같은 "별" 시리즈, 같은 구조).

## 현재 동작
- **풀기**: 수 가르기/모으기 다이어그램 1문제씩 → 0~9 키패드 입력 후 ‘입력’ 제출 → 별/도장 보상. 유형(가르기/모으기/섞기)은 사이드바에서 선택.
  단계(10까지 → 20까지 → 30까지)가 점점 열리는 사다리. 일일 미션·연속(streak) 기록.
  - **✏️ 풀이 연습장(S펜, 2026-07-13)**: 문제 옆(넓은 가로)·아래(세로)에 필기 캔버스. 펜/마우스만 그림+손가락 토글(팜 리젝션), 획 지우개·전체 지우기·접기·S펜 배럴버튼. 새 문제마다 자동 비움. (gobsem과 동일 `Pad`, 성경통독 필사 로직 이식.)
- **출력**: 유형·단계·쪽수·정답지 선택 → A4 한 장 9문제 학습지 미리보기 → 인쇄/PDF 저장. 매번 새 문제.
- **기록**: 도장 달력 + 통계.
- **부모 모드**: 제목 길게 누르기 → 비번. 단계 개방·미션/통과 수 조절·**기록 초기화(0으로, 2탭)**·비번 관리. (유형은 풀기 사이드바에서 선택)

## 구조 한눈에 (index.html 섹션)
1. `STAGES`/상수  2. `Store`(localStorage+메모리 폴백)  3. `makeProblem`(문제 생성)
4. `bondSVG`(다이어그램, play/print/answer)  5. 풀기  6. 단계/진행도  7. 축하 모달
8. 기록(도장)  9. 출력 학습지  10. 부모 모드  11. 시작
- 문제 단위: `{W,A,B,style,blank}`, 항상 `A+B===W`, 부분 ≥1.
- 정답 = 비운 칸의 실제 값(`answerOf`).
- 저장키 `byeolsugarugi_save_v1`. 보상 구조·달력은 byeol-sudoku와 동일 계열.

## 주의할 점
- **외부 라이브러리 없이 단일 파일** 유지(오프라인).
- 코드를 바꾸면 `sw.js` 캐시 버전(`byeolsugarugi-vN`)을 올려야 태블릿에 새 버전이 보임.
- localStorage는 반드시 `Store` 래퍼로.
- 다이어그램·학습지는 **자체 디자인** — 외부 학습지 브랜딩 복제 금지(문제는 새로 생성하므로 무관).

## 검증(이번 작업에서 한 것)
- 생성기 8만 회: `A+B===W`·범위·정답 정합성 전부 통과.
- jsdom 20개 테스트: 초기화·정답 기록·오답 미기록·단계 해금·출력(정답지 포함/제외)·부모 모드 통과.
- 로컬 서버로 모든 자산 200 확인. (`python3 -m http.server`)
