## 1. 문제 생성 (수 가르기·모으기)
- 단위: `{ W, A, B, style:'split'|'merge', blank:'top'|'left'|'right' }`, 항상 `A+B===W`, 부분 ≥1(학습지처럼 0 미사용).
- `split`(가르기): 위 W 주고 아래 한 칸 비움 / `merge`(모으기): 아래 두 수 주고 위(합) 비움.
- `shuffle`(빈칸 섞기): 가르기에서 좌/우 빈칸 랜덤.
- 직전 문제와 동일 회피. 모두 끄면 가르기로 폴백.

## 2. 다이어그램 — 전부 인라인 SVG (`bondSVG`)
- 화면(play)·인쇄(print)·정답(answer) 3모드. 박스+화살표(마커)+숫자 모두 SVG → 인쇄 시 벡터라 선명.
- play: 빈칸은 점멸 강조, 입력값 표시, 정답=초록/오답=빨강.

## 3. 풀기 모드
- 숫자판 1~단계max. **한 번 탭 = 즉시 채점**. 맞으면 별비 + 자동 다음, 틀리면 흔들림 + 재시도.
- 단계 사다리: `5까지(2~5) → 9까지(2~9) → 10(보수) → 십몇(11~19)`. `levelTarget`(기본 20) 달성 시 다음 해금.
- 일일 미션(`dailyTarget` 기본 10) 완료 시 도장+연속(streak), 마일스톤에서 축하 모달.

## 4. 출력 모드
- 유형(가르기/모으기/섞기)·단계(또는 섞기)·쪽수(1~10)·정답지 포함 선택 → "미리보기 만들기" → `window.print()`.
- A4 세로 한 장 = 3×3 = 9문제. 헤더(별 수가르기 + 월/일/이름 + 페이지번호) + "□ 안에 알맞은 수를 쓰세요." + 자체 푸터.
- `@media print`로 UI 숨기고 `#printArea`만, 페이지마다 분리. 디자인은 **자체 제작**(외부 브랜딩 복제 X).

## 5. 부모 모드
- 제목 길게 누르기 → 비번(미설정 시 생성). 토글/스텝퍼로:
  단계 "여기까지 열어주기", 유형(가르기/모으기/섞기) on·off, 오늘의 미션 수, 단계 통과 문제 수, 비번 변경/초기화.
- 설정은 `save.settings`에 저장. 부모 모드에서 푼 문제는 기록에 반영 안 됨.

## 6. 검증 (자세히)
- **생성기**: 4단계 × 5옵션셋 × 4000회 = 8만 회 → `A+B===W`·부분≥1·W범위·정답·style/blank 정합성 0 실패.
- **jsdom 20 테스트**: 초기화(SVG·패드·단계0), 정답→점수+100/별/오늘기록, 오답→미기록, levelTarget→maxLevel↑,
  출력 4시트(문제2+정답2)·시트당 9문제·문제 bond 숫자2개(빈칸)·정답 bond 숫자3개·인쇄버튼 활성, 부모모드 함수 존재.
- **로컬 서버**: index/manifest/sw/아이콘3 전부 200·올바른 content-type.
