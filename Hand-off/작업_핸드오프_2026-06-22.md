## 1. 문제 생성 (수 가르기·모으기)
- 단위: `{ W, A, B, style:'split'|'merge', blank:'top'|'left'|'right' }`, 항상 `A+B===W`, 부분 ≥1(학습지처럼 0 미사용).
- `split`(가르기): 위 W 주고 아래 한 칸 비움 / `merge`(모으기): 아래 두 수 주고 위(합) 비움.
- `shuffle`(빈칸 섞기): 가르기에서 좌/우 빈칸 랜덤.
- 직전 문제와 동일 회피. 모두 끄면 가르기로 폴백.

## 2. 다이어그램 — 전부 인라인 SVG (`bondSVG`)
- 화면(play)·인쇄(print)·정답(answer) 3모드. 박스+화살표(마커)+숫자 모두 SVG → 인쇄 시 벡터라 선명.
- play: 빈칸은 점멸 강조, 입력값 표시, 정답=초록/오답=빨강.

## 3. 풀기 모드
- 숫자판 1~단계max. **한 번 탭 = 즉시 채점**. 맞으면 별비 + 자동 다음, 틀리면 흔들림 + 재시도.
- 단계 사다리: `5까지 → 9까지 → 10까지 → 20까지 → 30까지`(min 2, max만 증가). `levelTarget`(기본 20) 달성 시 다음 해금. (※ 아래 "추가 작업"에서 초기 4단계 → 5단계로 확장)
- 일일 미션(`dailyTarget` 기본 10) 완료 시 도장+연속(streak), 마일스톤에서 축하 모달.

## 4. 출력 모드
- 유형(가르기/모으기/섞기)·단계(또는 섞기)·쪽수(1~10)·정답지 포함 선택 → "미리보기 만들기" → `window.print()`.
- A4 세로 한 장 = 3×3 = 9문제. 헤더(별 수가르기 + 월/일/이름 + 페이지번호) + "□ 안에 알맞은 수를 쓰세요." + 자체 푸터.
- `@media print`로 UI 숨기고 `#printArea`만, 페이지마다 분리. 디자인은 **자체 제작**(외부 브랜딩 복제 X).

## 5. 부모 모드
- 제목 길게 누르기 → 비번(미설정 시 생성). 토글/스텝퍼로:
  단계 "여기까지 열어주기", 유형(가르기/모으기/섞기) on·off, 오늘의 미션 수, 단계 통과 문제 수, 비번 변경/초기화.
- 설정은 `save.settings`에 저장. 부모 모드에서 푼 문제는 기록에 반영 안 됨.

## 추가 작업 (같은 날, 배포 후 피드백 반영)

배포 후 실제 화면을 보고 받은 피드백 3건 수정 → 커밋·push·라이브 반영 확인.

1. **다이어그램 네모(상자)가 안 보이던 버그** (`bondSVG`)
   - 원인: rect 너비/높이를 `b.w`/`b.h`(per-position 객체엔 없는 값)에서 읽어 width/height 미지정 → 상자 전부 안 보이고 숫자·화살표만 떴음.
   - 수정: `BOX.w`/`BOX.h` 사용. jsdom으로 rect 너비 74·높이 64·빈칸 blankpulse 검증. (캐시 v1→v2)
2. **가로/넓은 화면에서 내용이 위로 붙음** → `body justify-content: safe center`(출력 모드는 flex-start). 짧으면 세로중앙, 길면 위에서부터(잘림 방지).
3. **난이도 사다리 재정비**: 최종 **3단계** `10까지 → 20까지 → 30까지`(min 2 고정, max만 증가).
   - (경위: 초기 4단계[5까지/9까지/10/십몇] → 5단계로 확장 → 사용자 요청으로 5까지·9까지 제거. 10까지가 2~10을 포함해 작은 수 연습 유지.)
   - 숫자판 열 수 자동 조정: `max≤5→max, ≤20→5, 그 외 6열`. (캐시 v2→v3→v4)
   - **안전 클램프** 추가: `loadSave`·`newProblem`에서 `maxLevel`/`ui.level`을 단계 수 범위로 보정 → 단계 수 줄여도 기존 진도 데이터로 크래시 안 남(jsdom 검증).
   - 더 늘리려면 `STAGES`에 한 줄. 예) `{ key:"s50", label:"50까지", min:2, max:50 }`.
4. **입력 방식 변경: 답 후보 N버튼 → 0~9 키패드 직접 입력** (30까지에서 버튼 30개라 과함).
   - `buildPad`(1..max 가변) 제거 → `buildKeypad`(고정 0~9 + 지우기, 11키). 어떤 범위든 버튼 12개 이하.
   - `tapKey` **prefix 자동채점**: `next=entry+digit` → `Number(next)===답` 정답 / `String(답).startsWith(next)` 다음 자리 대기 / 그 외 즉시 오답·초기화. 두 자리 답(예 17)도 매끄럽게, 답 될 수 없는 입력은 바로 피드백.
   - 빈 네모 탭 → 입력 초기화. `ui.value(number)` → `ui.entry(string)`.
   - (손글씨 인식은 오프라인 단일파일에서 비현실적이라 키패드 채택.) jsdom 12체크 통과. (캐시 v4→v5)
5. **키패드 ‘입력’ 키 + 풀기 화면 유형 선택** (피드백 반영, 캐시 v5→v6)
   - 키패드 빈칸 제거: `지우기 | 0 | 입력`(12키). **명시적 제출**로 변경(숫자 입력 후 ‘입력’ 눌러 채점, 자동채점 폐지). 두 자리 제한.
   - **설정 모델 단순화**: `{split,merge,shuffle}` → `type:'split'|'merge'|'mix'` 하나. `optsForType()`로 생성옵션 변환. **loadSave에서 옛 booleans→type 자동 마이그레이션**.
   - **유형을 풀기 사이드바에서 직접 선택**(가르기/모으기/섞기) — 기존엔 출력/부모모드에만 있어 불편했던 점 해결. 부모 모드의 유형 토글은 제거(미션/통과 수·단계 개방만 유지).
   - jsdom 17체크 통과(키패드 12키·유형선택→style·명시적제출·두자리제한·마이그레이션).
